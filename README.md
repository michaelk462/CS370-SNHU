# CS370-SNHU
## What code was I given?
I was given 2 Python files to start: TreasureMaze.py and GameExperience.py.
**TreasureMaze.py** defines the maze environment as a matrix, handles movement, reward signals, and game state (win/loss)
**GameExperience.py** implements the replay buffer, which stores past episodes and samples random mini-batches for training.
The Jupyter Notebook **(“TreasureHuntGame.ipynb”)** also contained the maze definition, helper functions **(“show,” “play_game,” “completion_check”)**, the neural network architecture **(“build_model”)**, and a custom **“train_step”** function using TensorFlow's GradientTape.
## What code did I create?
I finished the two main parts that were left uncompleted on purpose:
**“play_game()”**: executes one game episode with the trained model in exploitation only mode, always choosing the highest valued valid action to step through the maze.
**“qtrain()”**: a complete deep Q-learning training loop implementing epsilon, greedy action selection (only valid moves are allowed), experience replay by **“GameExperience,”** target computation of the Bellman equation, a target network for training stability, epsilon decay, and early stopping when a 100% win rate is reached.
## What do computer scientists do and why does it matter?
Computer scientists create systems that can solve various challenges, from automating repetitive tasks to facilitating machine learning from experience. This project reflects a very small instance of it: instead of programming the pirate with a predetermined set of rules, I made a system that discovers the rules by itself through trial and error. Such a method applies to real-world issues in robotic navigation, medical diagnosis, and game AI. What truly makes computer science valuable is that the influence of its solutions expands: algorithms created in research labs become part of everyday tools, often in unexpected ways by the original developers.
## How do I approach a problem as a computer scientist?
I first figure out what the system already knows and what it needs to know. For this project, it was necessary to thoroughly examine the starter code before writing a single line of code, including the basis for the environment issuing rewards, how the replay buffer stored experiences, and the neural network's inputs and outputs. After that, I split the problem into smaller steps: action selection, training loop, and completion check verification. Whenever errors occurred, I thoroughly read the error message, identified the cause, and corrected them instead of starting over. That systematic approach is how I attempt to solve any technical problem.
## What are my ethical responsibilities to the end user and the organization?
In this game, there is hardly any ethical risk; habits are what matter most. I explained my programming in detail with inline comments so that anyone who reads it will not only understand the code but also the reasons behind each decision. Such a clear explanation is the ethical minimum. In fact, reinforcement learning application examples where the stakes are even higher, such as autonomous vehicles, recruitment software, and medical decision support, this principle has to be applied with a much larger sense of responsibility. An agent whose training is based on reward signals loaded with bias will be inclined to biased behavior. As a developer, it is my responsibility to examine the most appropriate reward function critically, consider who may be harmed in the event of a system failure, and verify that the system's behavior remains consistent across different starting conditions. The present project completion check, which ensures, during the validation process, that the agent can win from any position, is a simple illustration of this type of fairness testing.
